# 선수 이미지 적용 (넥슨 CDN 핫링크)

작성: 2026-07-10 · 등급: 보통(표출 전용, 데이터 비가역 변경 없음)

## 배경

리더보드가 선수를 이름 문자열로만 보여준다. 넥슨 Open API 문서의 Image 섹션이
`fco.dn.nexoncdn.co.kr` 의 선수 이미지 엔드포인트 4종을 공개하고 있어, 이를 붙여
리더보드·상세·비교탭의 시각적 식별성을 높인다.

문서에 적힌 4종을 리더보드 상위 40장 표본으로 전부 GET 해 실측한 결과, **문서와 실제가 다르다.**

| 경로 | 커버리지 | 규격 |
|---|---|---|
| `players/p{pid}.png` | 40/40 (100%) | 128×128 RGBA, 평균 23KB |
| `playersAction/p{spid}.png` | 25/40 (62%) | 128×128 RGBA, 평균 29KB |
| `playersAction/p{pid}.png` | 23/40 (57%) | 128×128 RGBA |
| `players/p{spid}.png` | **1/40 (2%)** | 대부분 403 (XML 에러) |

즉 **spid로 정적 얼굴 이미지를 받는 경로는 사실상 죽어 있다.** 100% 뜨는 건 pid 경로뿐이다.

## 실측 근거

이 설계가 의존하는 사실은 전부 라이브 CDN과 실제 `out/leaderboard.json` 으로 검증했다.

- **핫링크 차단 없음** — `Referer` 를 없애거나, Vercel 도메인, `example.com` 으로 바꿔도 전부 200.
- **캐시 헤더** — `cache-control` 없음. `ETag` 와 `Last-Modified` 만 내려온다.
- **`<img>` 태그는 CORS 불필요** — `access-control-allow-origin` 부재는 문제가 되지 않는다.
- **spid 구조** — 리더보드 1,817장 전량이 9자리이고 `season_id` 는 3자리. `pid = spid 뒤 6자리` 위반 0건.
- **선행 0 규칙은 실재한다** — 1,817장 중 **250장(13.8%)**, 유니크 pid 13개가 선행 0을 갖는다.
  `p000488.png` → 403, `p488.png` → 200. 제거하지 않으면 카드 일곱 장 중 하나가 조용히 깨진다.
- **투명 컷아웃** — 둘 다 colortype=6(RGBA), 네 모서리 alpha=0.
- **프레이밍 일관성** — portrait 6장 표본의 피사체 바운딩 박스가 x 4~124, y 4~127로 거의 동일.
  얼굴 중심이 (62, 68)로 캔버스 중심보다 살짝 아래.
- **`same_player` 182그룹 전부 내부 pid 일치** — 그룹 썸네일을 `cards[0].gk_sp_id` 에서 파생해도 안전.

## 목표

리더보드 행, 상세 패널, 동일 선수 비교 탭 세 곳에 선수 이미지를 노출한다.

## In-scope

- `src/gksave/render.py` (HTML/CSS/JS 템플릿) — **유일한 수정 대상 소스**
- `tests/test_render.py` — 렌더 결과 검증 추가

## Out-of-scope

- `out/leaderboard.json` 스키마 — `gk_sp_id` 하나로 두 URL이 전부 파생되므로 변경 없음
- `src/gksave/export.py` · `agg.py` · `meta.py` — 변경 없음
- 이미지 자체 호스팅 · 빌드 단계 · WebP 변환 · Pillow 의존성
- 시즌 엠블럼 이미지 (별도 엔드포인트, 이번 범위 아님)

## 핵심 결정

| 결정 | 근거 |
|---|---|
| 전량 CDN 핫링크 (자체 호스팅 안 함) | 저장소 증가 0바이트, 빌드 단계 없음. 핫링크 차단 부재를 실측. 자체 호스팅은 히어로용 유니크 spid 855개≈24MB를 git에 얹는다 |
| 썸네일은 `players/p{pid}` | 커버리지 100% → 실패 요청 0. 유니크 pid 235개뿐이라 정렬·검색 재렌더 시 브라우저 캐시 히트 |
| 히어로는 `playersAction/p{spid}` | 시즌 유니폼을 보여준다. 행 클릭 시 1장만 로드하므로 38% 폴백 비용이 무의미 |
| 썸네일에 액션샷을 쓰지 않음 | 초기 100행에서 403이 38개 발생하고, 36px 원형에서는 유니폼 차이가 보이지도 않는다 |
| pid 파생을 JS에 둠 (payload에 URL 미포함) | URL을 넣으면 JSON이 1,817 × ~120B ≈ 220KB 증가. 진행 중인 payload 압축 계획과 역행 |
| 상세는 히어로 스트립 추가, 그리드 유지 | 3열 재구성은 모바일 브레이크포인트까지 건드린다. 스트립은 기존 2열 그리드를 건드리지 않는다 |
| 플레이스홀더는 인라인 SVG data URI | 자기완결형 HTML 한 장이라는 `render.py` 기조 유지 (외부 의존 0) |
| 히어로 112px, 업스케일 없음 | 원본이 128px. 그 이상 키우면 흐려진다 |

## 상세 설계

### URL 파생

```js
const CDN = 'https://fco.dn.nexoncdn.co.kr/live/externalAssets/common';
const pidOf = spid => String(spid).slice(-6).replace(/^0+/, '');  // 선행 0 제거 — 250장이 걸린다
const portraitUrl = spid => `${CDN}/players/p${pidOf(spid)}.png`;
const actionUrl   = spid => `${CDN}/playersAction/p${spid}.png`;
```

### 폴백 체인

- 썸네일: `players/p{pid}` → 플레이스홀더
- 히어로: `playersAction/p{spid}` → `players/p{pid}` → 플레이스홀더

`onerror` 핸들러는 진입 즉시 `this.onerror = null` 로 자신을 끊어 무한 루프를 막는다.
플레이스홀더까지 실패할 수 없으므로(data URI) 체인은 반드시 종료된다.

### 썸네일

리더보드 행의 선수 이름 왼쪽, 그리고 비교탭 각 그룹 `<summary>` 왼쪽에 36px 원형.

- `loading="lazy" decoding="async" width="36" height="36"` — 레이아웃 시프트 방지
- `object-fit: cover; object-position: 50% 42%` — 실측한 얼굴 중심 (62,68)을 원 안으로 당긴다
- 투명 PNG이므로 어두운 원판 배경 + 골드 링 테두리(`--gold2`)를 깐다
- 비교탭 그룹 썸네일은 `cards[0].gk_sp_id` 에서 파생 (182그룹 전부 내부 pid 일치 확인)

### 히어로 스트립

상세 패널(`detailHtml`) 최상단, 기존 2열 그리드 **위**에 배치.

- 왼쪽: 액션샷 112px (`width="112" height="112"`, 업스케일 없음)
- 오른쪽: 선수명 · 시즌 · 강화단계 · 선방률 요약
- 모바일 640px 이하에서 세로 스택
- 행 클릭 시에만 DOM에 삽입되므로 초기 로드에 영향 없음

### 성능

- 초기 렌더는 100행이지만 lazy load라 뷰포트 내 15장 남짓(≈350KB)만 즉시 요청
- 유니크 pid 235개 → 정렬·검색으로 `tbody` 를 통째로 갈아끼워도 재요청 없이 캐시에서 나온다
- CDN에 `cache-control` 이 없어 재방문 시 304 왕복이 생긴다. 핫링크의 대가로 수용

## 테스트 전략

`build_html` 은 순수 함수이므로 `tests/test_render.py` 에서 렌더 결과 문자열을 검증한다.

- CDN 베이스 URL과 두 경로(`/players/p`, `/playersAction/p`)가 템플릿에 존재
- `loading="lazy"` 와 `this.onerror=null` 이 존재
- 플레이스홀더 data URI가 존재
- 기존 3개 테스트(구조·주입방지·빈 리더보드)가 계속 통과

pid 파생 로직은 JS 안에 있어 파이썬 테스트가 닿지 않는다. 그 대신 규칙 자체를
리더보드 1,817장 전량으로 검증했고(위반 0건), 선행 0 케이스 13개를 라이브 CDN으로 실측했다.
브라우저 동작(AC1~AC8, AC11)은 `/qa` 로 확인한다.

## 남은 리스크

- **넥슨이 핫링크를 차단하면 모든 이미지가 깨진다.** 폴백 체인 종단의 플레이스홀더가
  이를 흡수해 페이지는 실루엣으로 정상 렌더된다. 데이터·지표에는 영향이 없다.
- **오프라인에서 `out/index.html` 을 열면 이미지가 안 뜬다.** 플레이스홀더로 대체된다. 수용.
- **커버리지 62%는 40장 표본 추정치다.** 히어로 폴백이 어차피 이를 처리하므로
  실제 값이 달라도 동작은 바뀌지 않는다.
- CDN이 `cache-control` 을 주지 않아 재방문 304 왕복이 발생한다.
