# 계획: 상황별 키퍼 찾기 + 가성비 랭킹

작성: 2026-07-16 · plan-forge (심도: 보통~중대) · 대상: `agg.py`, `export.py`, `render.py`, tests

## 목표
리더보드에 (1) 거리존/상황별 선방률로 정렬·게이트하는 "상황별 키퍼 찾기"와
(3) 급여 대비 성능(GSAx/선방률)으로 정렬하는 "가성비" 뷰를 추가한다. 네 사이트만
가진 실제 경기 데이터(상황별 성능)를 전면에 세우는 게 목적.

## 조사로 확정된 사실
- 슬림 카드(정렬용, index.html 임베드)엔 save_pct·gsax_per_shot·info.salary·info.ovr 있음 → **가성비는 슬림만으로 가능**.
- 거리존/상황은 details.json(지연 로드)에 있어 슬림엔 없음 → 상황 정렬용 **요약을 슬림에 추가** 필요.
- 거리존(zones)은 슛수(분모) 있음. 상황(extras: 1대1·박스안/밖·연계)은 비율만·**분모 없음**.
  단 `agg.card_extras_all` SQL 이 분모(in_n·out_n·u_n·a_n)를 **이미 계산** — 출력만 추가하면 게이트 가능.
- 게이트200 카드 2095장. 원·중거리는 전 카드 표본 충분, 근·초근은 일부 얇음 → 상황 표본 게이트 필수.
- 급여 없는 카드 2장(가성비 제외).

## 핵심 결정 (UI)
- **"지표" 드롭다운으로 선방률 컬럼을 문맥 전환**(새 컬럼 안 늘림). 선택지: 선방률(기본)/
  원거리·중거리·근거리·초근거리/1대1·박스안·박스밖·연계/가성비. 고른 지표로 컬럼 값·헤더·정렬 전환.
- 상황별은 **표본 게이트(그 상황 슛 ≥30)** — 미달 카드 제외(노이즈 방지).
- **가성비**: 기본 급여 대비 GSAx(gsax_per_shot/salary), 작은 토글 `선방률 기준`으로 급여 대비 선방률(save_pct/salary) 전환. 급여 null 제외.
- GSAx 컬럼·초근제외 토글은 유지. 상단 "참고용" 경고는 상황별에도 그대로 적용(raw 교란 잔존).

## In-scope
- `agg.py`: 상황 분모 4종(in_pen_shots, out_pen_shots, unassisted_shots, assisted_shots) 출력 추가.
- `export.py`: 슬림 카드에 `sit` 요약 — 거리 4구간 {pct,shots} + 상황 4종 {pct,shots}. (~16숫자/카드)
- `render.py`: "지표" 드롭다운(선방률 컬럼 전환) + 상황 표본 게이트 + 가성비(GSAx/선방률 토글).
- 테스트: 상황 정렬·게이트·가성비 계산·기준 토글·기본 회귀.

## Out-of-scope (로드맵)
- 슛타입별(감아차기 등 — 종류 많고 표본 얇음), 상황별 신뢰구간, 상황 게이트 값 사용자 조절.

## 수용 기준
1. "지표=원거리 선방률" 선택 시 선방률 컬럼이 원거리 선방률(+슛수)로 바뀌고 그 값 내림 정렬, 원거리 슛<30 제외.
2. 1대1/박스안/박스밖/연계도 동일하게 동작(분모로 게이트).
3. "지표=가성비" 시 급여 대비 GSAx 내림 정렬, `선방률 기준` 토글로 급여 대비 선방률로 전환·재정렬.
4. "지표=선방률(기본)" 복귀 시 원래 컬럼·정렬(회귀 없음). 이름·국가/클럽·급여·강화·게이트 필터와 병행.
5. 급여 없는 카드는 가성비 순위에서 빠진다.
6. 테스트 통과.

## 채택된 가정
- 상황 표본 게이트 = 30슛(원·중은 사실상 전부 통과, 근·초근·상황에서 노이즈 컷). 값은 상수로 두고 조절은 로드맵.
- 상황별 선방률 표기는 pct + "(N슛)"; 신뢰구간은 생략(슬림 절약).
- 가성비 값은 순위용 — 절대값보다 순서가 의미(비율 크기 자체는 표기 최소).

## 남은 리스크
- 상황별도 보정 안 된 raw → "참고용" 경고 적용, 게이트로 노이즈만 완화.
- 슬림 소폭 증가(상황 요약) — details 분리 이득 대비 미미.
- "지표" 드롭다운이 헤더정렬과 상호작용: 특수지표 활성 시 헤더정렬 대신 그 지표로 정렬(선방률 컬럼만 전환, 나머지 컬럼 헤더정렬은 그대로 두되 특수지표가 우선).

## 작업
1. `agg.py` card_extras_all 분모 출력. 검증: extras 에 *_shots 4키.
2. `export.py` 슬림 `sit` 요약(거리·상황 {pct,shots}). 검증: 슬림 카드에 sit, details엔 영향 없음.
3. `render.py` "지표" 드롭다운 + sortMetric 상태 + 상황 게이트 + 가성비(GSAx/선방률 토글). 검증: 프리뷰.
4. 테스트: node 정렬/게이트/가성비 계산, 마크업 존재.
5. export 재생성 → 로컬 http 헤드리스 검증(상황·가성비 정렬) → 커밋(feat + chore).
