# 계획: GK 리더보드 페이지 개편 — 탭·지표설명·경량화

작성: 2026-07-08 · plan-forge (심도: 보통) · 대상 파일: `src/gksave/render.py` 단일

## 목표
FC온라인 GK 리더보드 페이지를 (1) 지표 이해 가능 (2) 원하는 정보 빠른 도달 (3) 빠른 로딩으로 개편.

## In-scope
- 탭 3개: `[리더보드]` · `[동일 선수 비교]` · `[지표 설명·사용법]` (vanilla JS 토글).
- 리더보드 기본 상위 100명 + "더 보기"/"전체 보기". 검색어 있으면 전체 1,544장에서 필터(캡 무시).
- 지표 설명 탭: 핵심 3종(선방률·GSAx·표본/게이트)+사용법 상시 펼침, 세부지표 접이식(`<details>`).
- 강화 효과(grade_effect) 한 줄 + 주의 라벨을 **상단에 상시 노출**(탭 무관).
- 경량화: CDN 폰트 `<link>` 제거(시스템 폰트 폴백) + 초기 DOM 상위 100행 제한.

## Out-of-scope (로드맵)
- 드릴다운 상세 별도 파일 분리(자기완결 포기), URL 해시 라우팅/딥링크, 가상 스크롤 라이브러리,
  서버측 정렬/검색, `export.py`/집계 로직 변경.

## 수용 기준
1. 로드 시 탭 3개, 기본 활성 `[리더보드]`.
2. 검색어 없을 때 리더보드 정확히 100행, "더 보기"로 다음 100행 추가.
3. 존재하는 선수명 검색 시 상위 100위 밖(예 300위)이라도 결과에 노출.
4. `[동일 선수 비교]` 탭 클릭 시 스크롤 없이 비교표 즉시 표시.
5. `[지표 설명]` 탭에 핵심 지표 펼침 + 세부 접이식.
6. HTML에 외부 URL 참조 0건(자기완결 유지).
7. 행 클릭 드릴다운·정렬·동일선수표 회귀 없음.
8. 강화 효과 한 줄이 상단에 상시 보임.

## 핵심 결정
- 탭 = 순수 CSS/JS `display` 토글(외부 라이브러리 없음).
- 상위 100 캡은 "표시"만, 데이터는 전량 임베드(단일 HTML 유지). 검색은 전체 대상.
- CDN 폰트 제거 → Pretendard→시스템 폰트 미세 변화 수용.

## 채택된 가정
- 강화 효과 한 줄 = 상단 배너(상시) + 상세는 지표 설명 탭.
- 더 보기 100행 단위 + "전체 보기".
- 주의 라벨 = 상단 얇은 배너 1줄 + 전문은 지표 설명 탭.
- 탭 상태는 새로고침 시 리더보드로 초기화(해시 없음). 모바일 탭 바 가로 유지.

## 작업 (전부 `src/gksave/render.py`)
1. 탭 뼈대 + CSS + `showTab()`; 기존 마크업을 3패널로 재배치. 검증: 탭 3개 전환(수용 1,4).
2. CDN 폰트 `<link>` 제거, 시스템 폰트 스택 유지. 검증: 외부 URL grep 0건(수용 6).
3. 리더보드 상위 100 캡 + 더보기; 검색 시 캡 무시. 검증: 기본 100행 + 100위밖 검색 노출(수용 2,3).
4. 지표 설명 탭 정적 콘텐츠(핵심 펼침 + 세부 details) + 강화효과 상단 한 줄. 검증: 수용 5,8.
5. 회귀 확인 + `gksave export --gate 50 --out out` 재생성. 검증: 수용 7.
